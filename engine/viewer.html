Web Viewer

The web viewer is a purely read‑only consumer of Replay data.

Purpose

The viewer allows a human user to: - inspect a played game - replay the evolution of the board - understand agent decisions - inspect learned vs heuristic rationales

The viewer is designed for: - debugging - qualitative evaluation - demonstration - analysis

Non‑Responsibilities

The viewer does NOT: - execute game logic - modify game state - compute rewards - infer rationales - interact with the environment - interact with the policy

The viewer never accesses: - the environment - the policy - PPO internals

Data Source

The viewer consumes only Replay JSON files.

All information shown in the viewer must be derivable from: - Replay - ReplayStep - GameState - Decision (optional)

The viewer must remain compatible with: - replays generated by RLRunner - replays generated by neutral runners

Design Principle

The viewer is presentation only.

If the viewer needs information that is not present in the Replay, that information must be added to the Replay schema, never computed inside the viewer.
